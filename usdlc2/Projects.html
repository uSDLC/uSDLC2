Projects

All the documentation, instrumentation and support files are saved with the project source files. To unify the software development lifecycle, all relevant data needs to be recorded together for availability.

Adding a Project

The same procedure is used to create a project, add a new project or add an project with an existing uSDLC component. The left-most drop-down on the uSDLC tab is used to select known projects. The last item, new..., will open a dialog box. Give the project a name and a base path.

Base Path

This is the path relative to the directory holding uSDLC. The text box uses auto-complete to aid in finding existing projects or the base directory for a new project. A project name is generated from the final directory in the base path.

Project Name

Different installations can use different names for a project. Only the base path is used to connect the projects internally and externally.

Artifacts Added to Project

projects.ini

First the uSDLC2 project has a local directory that is not added to version control. In that directory is a projects.ini file. There is one entry per project of the form:

Project_Name=base:../path/from/uSDLC2

Any underscores in project name will be displayed as spaces on the UI. The path is from inside uSDLC2 - so one deeper than the one specified in the UI.

usdlc2

Only one directory is important to retaining the project design, instrumentation and test system, usdlc2.

The usdlc2 directory should be checked in to version control with the source.

document.css

This file is copied from uSDLC2 during project creation and can be updated to provide different formats for different projects. Every HTML file refers to it so that formatting will work inside uSDLC2 or when viewing the documents directly in a browser.

Index.html

An empty Index.html file is created for new projects. It is used to reference and create new documents for the project. It has no special flavour other than being the first document, so any html format or layout can be used.

Documents

All other HTML files in the directory (or ones below it) represent documents for system. uSDLC2 does not enforce structure for documentation, but is is easier to maintain few files than many small ones with lots of cross-references.

gen

The document files contain most of the gwt, bridge code, data and meta-data. When instrumentation happens any updates are extracted as separate files in the gen directory.

Exclude this directory from version control as any information it holds is rebuilt when needed.
