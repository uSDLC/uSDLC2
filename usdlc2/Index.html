uSDLC2 unifying the software development lifecycle

uSDLC2 uses minimalist documentation and instrumentation to connect all the tasks for a software development project.

Projects

uSDLC2 artefacts are stored with your project source in version control. The UI provides facilities to connect existing projects to uSDLC2 - or to create new ones.

Ref: app.coffee

Documents

Within a project, documents are stored with other artefacts in a subdirectory called usdlc2. Creating new documents is as easy as creating new hyper-links and following them.

Instrumentation

Instrumentation is the link between the code and the design. The nature of the instrumentation depends on the target group. It will be natural language for features and examples, or code for internal development designs.

Developer

uSDLC2 provides tools for a developer to link instrumentation to the running code-base.

Metadata

Each section can have metadata attached that aids in navigation, workflow and project management.
