Introduction (screencast)

uSDLC2 generates a set of html documents in the usdlc2/ directory of your project. These can be reviewed in a browser without additional software - and contain all documentation and instrumentation details and code.

uSDLC2 uses the open source CKEditor for simple WYSIWYG editing and CodeMirror for source code and data editing.

First Page Load

After a first install, uSDLC2 will be the current project, with Index as the current document. Whenever the page is reloaded, the current project, document and page location are restored.

The tabs along the top break down editing functionality. All tabs but the last are core WYSIWYG editor functions.

The uSDLC Tab

The left box displays the current project. Select it to change project or add a new one. When switching projects the last document and location are restored.

The second box displays the current document. Switch documents here. New documents are created from hyperlinks and will display in the list. You can also switch to the new document from the hyperlink by holding shift down when selecting it.

The next button displays a structured table of contents for the current document.

The button bar to the right implement uSDLC2 specific functionality. Hover over each button to see it's function and short-cut key.

Hyperlinks

Highlight any phrase and click the link button to create a link to a new or existing document. More general hyperlinks can be created from the Insert tab. Once a link is created it can be modified by double-clicking or followed by shift-clicking.

Instrumentation

The target us used to add instrumentation instructions, code or data to the document. Anything added here can be accessed and used to connection to the production system or parts thereof.

Bridges

Instrumentation is a visible instruction to exercise the production system. The bridge code takes this instrumentation and makes it happen.

The Bookshelf

The bookshelf is a way to access all the files in the project.

Play

Once instrumentation and bridging are complete, press the play button to instrument the current and all inner sections.

A Terminal

This is a standard terminal to the server containing the project. All non-UI shell commands are available.
