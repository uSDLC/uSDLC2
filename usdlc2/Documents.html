Documents

uSDLC2 saves all documents as standard HTML files. Instrumentation details and bridge code are held within the document and extracted for instrumentation.

The Index

When adding a new project, an index file is referenced automatically. Except for very small projects, this will link to documents serving different parts of the system.

I will typically use the project name as the page title and use heading 2 for each section. Highlight each heading and press Alt-L to turn it into a hyperlink to a related document. Shift-click to follow it to the new page.

Sections

uSDLC2 documents are free-form. Instrumentation uses sections to define which parts to run and which bridge code to reference. Sections are defined using headings, where the heading number provides a tree structure.

Editing

uSDLC2 uses CKEditor for documents, dividing up the provided functionality into tabs. All the standard WYSIWYG features are provided, along with uSDLC2 specific ones in the last tab. A uSDLC2 document is a HTML file that can be clearly displayed in a browser without running the application. Editors can manipulate data inserted or maintained by uSDLC as they can any other document element.

Creating links between documents

CKEditor provides a general link in the Insert tab. The Link action is a shortcut on the uSDLC tab, the context menu or by typing alt-L. Highlight some text first then use the above methods to create a link. The document will have the same name as the link text - with spaces and other non-word characters converted to underscores. If the document exists it is opened, otherwise a new document is created.

If you do not highlight text the a dialog will load displaying all documents from all projects. Select one to insert it.

Navigation

The Address Bar

The address bar can be used for all the navigation elements - Project, Document and Section.

http://localhost:9009/uSDLC2/Documents?edit#Navigation

This example assumes a standard local install of uSDLC2. It will open a page on uSDLC2. Each project is referenced by name as uSDLC2 can convert this to a path to the project. The HTML document comes next. If the document doesn't exist, it is created when it is given content.®

Each project remembers the last Document and Section displayed - and will return there on a reload if they are not on the command-line.

http://localhost:9009/uSDLC2?edit

Normally you will only need to bookmark to the project level - which will return you to where you last left off.

Between Documents

The document to edit can be selected from a link by shift-clicking. The Index page will typically have links to everything (if you put them there).

The second drop-down on the uSDLC tab can be used to select the current document in the current project.

When changing projects the last document editing on this browser by this user will be loaded.

The address line can have the document added after the project - as above. Note that spaces are represented by underscores.

Finally links can be created anywhere that reference other document.

Saving Documents

uSDLC automatically saves a document in pauses in typing or change of focus between windows. A Saved message will display to the right of the editor tabs.

A save operation includes merging of other data windows into the main document then creating a difference document between the expected document on disk and the changes. If the document on disk has been changed by some external method (such as a version control update) then it will not be possible to save the document without merging the changes.
