Production Server

General
The 'go' script will start a production server on command. A production server logs exceptions and continues for new connections. It allows the setting or a running port and default user on the command line.

Server Options

When starting a server using uSDLC2/go server, you can add command-line options:

  1. port=9009:  set the port uSDLC2 listens on. Defaults to 9009.
  2. user=Guest: Set the default user that does not require log-in. Default is 'Guest' - a visitor who can see little and change less. In a development environment you can set this to your user name to save logging in every time the server restarts. The user name can be full name, email address or initials. Use quotes if the name includes spaces.

~/dev/uSDLC2/go server port=9011 "user=Paul Marrington"
